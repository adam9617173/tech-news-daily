嚴重 Linux CopyFail 漏洞！美國政府：5/15 前必須修補

2026-05-05 來源：TechCrunch

一個影響幾乎所有 Linux 版本的嚴重安全漏洞已被公開，安全研究人員發布了允許攻擊者完全控制受影響系統的 exploit code。美國政府已將其列為已知被利用的漏洞，正在被積極用於惡意攻擊。

這個代號為「CopyFail」的漏洞（官方編號 CVE-2026-31431），影響 Linux kernel 7.0 及更早版本。雖然 3 月下旬已向 Linux kernel 安全團隊披露並在一週後修補，但補丁還沒有完全下放到眾多依賴受影響 kernel 的 Linux 發行版。

根據安全公司 Theori 的發現，CopyFail 漏洞可在多個廣泛使用的 Linux 版本上運作，包括 Red Hat Enterprise Linux 10.1、Ubuntu 24.04 LTS、Amazon Linux 2023 和 SUSE 16。該網站稱同一個簡短的 Python script 「root 了自 2017 年以來發布的所有 Linux 發行版」。

如果被利用，常規的有限訪問用戶可以獲得受影響 Linux 系統的完全管理員權限。數據中心伺服器被成功入侵後，攻擊者可以存取每個應用程式、伺服器和大批企業客戶的資料庫。CISA 已命令所有民間聯邦機構在 5 月 15 日前修補任何受影響的系統。

💡 C寶的見解

這是一個「十年一遇」等級的漏洞。自 2017 年以來的所有 Linux 發行版都中招，而且 exploit code 已公開。未來幾天，所有 Linux 伺服器管理員都要加班補丁了。這會大幅增加 IT workload。
